Resources

Event/event?when

An Event is a time-limited sale of zero or more Product items.

Methods

GET

Retrieve a collection of all active/current Event instances.

request query parameters
parametervaluedescription

when

string (repeating)

One of:

  • past
  • current
  • upcoming

A comma-separated list of time period names to filter events on.

available response representations:

  • Event Collection (application/json)

resource-wide template parameters
parametervaluedescription

string

The unique Event identifier.

GET

User Authorization/auth

A user authorization is the analog to logging into the system on behalf of an existing system User in order to perform sensitive operations on their behalf.

Methods

POST

Login to the system on behalf of an existing system User. Upon successful login, an HTTP cookie will be supplied in the response to allow further requests on behalf of the logged-in user.

acceptable request representations:

  • Login Request (application/json)

available response representations:

  • User Authorization Entity (application/json)

DELETE

Destroy an existing user session and log the system User out of the system.

User/user

A system User represents an account that enables the holder to perform certain activities, including purchasing Products.

Methods

POST

Add a new User to the system. NOTE: The credit property will be ignored when creating or updating users.

acceptable request representations:

  • Partial User Entity (application/json)

available response representations:

  • Full User Entity (application/json)

User Orders/user/{user_id}/order

The collection of all orders that a user has ever placed.

resource-wide template parameters
parametervaluedescription

user_id

string

The unique User identifier.

Methods

GET

Retrieve the collection of all orders that a user has ever placed.

response header parameters
parametervaluedescription

Cookie

string (required)

The HTTP cookie provided by the server when logging a system User into the system.

available response representations:

  • Order Collection (application/json)

POST

Submit a new Order for this user. An order is intially created with one or more product items, and is assigned an expiration date. Then an order can be continually updated with more items, which updates the expiration date. Once the expiration date passes, the incomplete order is destroyed and will cease to exist.

Finally, once both payment and addresses information is submitted, the order is considered complete, at which point is no longer contains an expiry date.

request header parameters
parametervaluedescription

Cookie

string (required)

The HTTP cookie provided by the server when logging a system User into the system.

acceptable request representations:

  • Partial Order Entity (application/json)
response header parameters
parametervaluedescription

Cookie

string (required)

The HTTP cookie provided by the server when logging a system User into the system.

available response representations:

  • Full Order Entity (Pending) (application/json)
response header parameters
parametervaluedescription

Cookie

string (required)

The HTTP cookie provided by the server when logging a system User into the system.

available response representations:

  • Full Order Entity (Placed) (application/json)
response header parameters
parametervaluedescription
